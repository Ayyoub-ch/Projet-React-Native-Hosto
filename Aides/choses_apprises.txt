Mémo Firebase (Simple & Clair)

1. Qu’est‑ce que Firebase ?

Firebase est une plateforme de Google qui fournit : - Authentification
(Google, email/mot de passe, etc.) - Bases de données : Firestore
(NoSQL) - Stockage (images) - Hébergement - Analytics

Il fonctionne parfaitement avec React Native + Expo sans configuration
native compliquée.

3.  Clique Créer base de données

Note : focus dans React Navigation
1️⃣ À quoi ça sert

Le focus permet de savoir quand un écran devient actif à l’écran. C’est utile quand tu veux :

Mettre à jour l’UI selon l’écran affiché.

Charger des données uniquement quand l’utilisateur voit l’écran.

Activer ou désactiver des boutons globaux (comme ton bouton hamburger).

2️⃣ Exemple concret

Supposons que tu as un bouton hamburger global, mais tu veux qu’il n’apparaisse pas sur les pages de login et signup.

import { useState } from "react";
import { TouchableOpacity, Text } from "react-native";
import { useNavigation, useFocusEffect } from "@react-navigation/native";

function ProfilScreen() {
  const [showMessage, setShowMessage] = useState(false);

  // useFocusEffect s’exécute à chaque fois que l’écran devient actif
  useFocusEffect(
    React.useCallback(() => {
      console.log("ProfilScreen est affiché !");
      setShowMessage(true);

      return () => {
        console.log("ProfilScreen n'est plus affiché !");
        setShowMessage(false);
      };
    }, [])
  );

  return (
    <>
      {showMessage && <Text>Bienvenue sur Profil !</Text>}
    </>
  );
}


Explication :

useFocusEffect est appelé quand l’écran devient actif.

Ici, on affiche un message seulement quand l’écran est visible.

Quand on quitte l’écran, le cleanup dans le return se déclenche pour masquer le message.

3️⃣ Résumé

Focus = écran visible pour l’utilisateur.

Permet de réagir dynamiquement selon l’écran actif.

Très utile pour des boutons globaux, chargement de données, analytics, etc.

NavigateAndClose (Naviguer puis fermer un drawer)
1️⃣ Définition

"Navigate and Close" est un pattern courant dans les applications utilisant un drawer ou un modal :

On veut naviguer vers un nouvel écran et, en même temps, fermer le drawer ou modal pour que l’utilisateur arrive directement sur le contenu cible.

Dans React Native avec DrawerLayoutAndroid ou @react-navigation/drawer :

Navigate : permet de changer d’écran (navigation.navigate("ScreenName"))

Close : permet de fermer le drawer (drawerRef.current.closeDrawer() ou navigation.closeDrawer() si on utilise le Drawer Navigator officiel)

2️⃣ Particularités

Ordre important :

Il faut d’abord naviguer, puis fermer le drawer, ou parfois d’abord fermer le drawer, puis naviguer, selon le comportement attendu.

Si on ferme le drawer après la navigation, on évite que le drawer reste ouvert sur le nouvel écran.

Accessibilité du drawer :

Avec DrawerLayoutAndroid (API native Android), il faut passer une référence (ref) pour fermer le drawer.

Avec @react-navigation/drawer, navigation.closeDrawer() fonctionne directement depuis l’écran ou le composant enfant.

Éviter les erreurs :

Si la référence n’existe pas ou que le drawer est déjà fermé, l’appel à closeDrawer() peut échouer.

Vérifier avec if (drawerRef.current) avant de fermer.

3️⃣ Exemple pratique avec DrawerLayoutAndroid

DrawerContent.js :

import React from "react";
import { View, Text, Pressable } from "react-native";
import { useNavigation } from "@react-navigation/native";

const DrawerContent = ({ drawerRef }) => {
  const navigation = useNavigation();

  const navigateAndClose = (screenName) => {
    navigation.navigate(screenName);     // 1️⃣ Naviguer vers l’écran cible
    if (drawerRef && drawerRef.current) {
      drawerRef.current.closeDrawer();   // 2️⃣ Fermer le drawer
    }
  };

  return (
    <View style={{ flex: 1, padding: 20, backgroundColor: "#222" }}>
      <Pressable onPress={() => navigateAndClose("Profil")}>
        <Text style={{ color: "#fff", fontSize: 18, marginVertical: 10 }}>Profil</Text>
      </Pressable>

      <Pressable onPress={() => navigateAndClose("Progression")}>
        <Text style={{ color: "#fff", fontSize: 18, marginVertical: 10 }}>Progression</Text>
      </Pressable>
    </View>
  );
};

export default DrawerContent;

4️⃣ Exemple pratique avec @react-navigation/drawer

Si on utilise le Drawer Navigator officiel :

<Drawer.Screen name="Profil" component={Profil} />


Dans le contenu du drawer :

const navigation = useNavigation();

<Pressable onPress={() => {
  navigation.navigate("Profil");
  navigation.closeDrawer();  // Fonction intégrée
}}>
  <Text>Profil</Text>
</Pressable>


Ici, pas besoin de ref, React Navigation gère tout.

5️⃣ Avantages

Expérience utilisateur fluide : l’utilisateur arrive directement sur le nouvel écran sans que le drawer reste ouvert.

Peut être combiné avec un state drawerOpen pour changer l’icône du bouton hamburger en menu ou close.

Fonctionne à la fois avec DrawerLayoutAndroid et le Drawer Navigator.

| Point                  | DrawerLayoutAndroid                     | Drawer Navigator           |
| ---------------------- | --------------------------------------- | -------------------------- |
| Fermer le drawer       | `drawerRef.current.closeDrawer()`       | `navigation.closeDrawer()` |
| Naviguer vers un écran | `navigation.navigate("ScreenName")`     | Même chose                 |
| Utilisation combinée   | Naviguer puis fermer                    | Naviguer puis fermer       |
| Sécurité               | Vérifier que `drawerRef.current` existe | Pas nécessaire             |



Pour résumer, "Navigate and Close" est un pattern essentiel pour une navigation fluide 
dans les apps avec drawer. Que tu utilises DrawerLayoutAndroid ou le Drawer Navigator officiel, 
l’idée reste la même : naviguer vers l’écran cible et fermer le drawer pour une UX optimale.
